## 42.4. Dados globais em PL/Tcl [#](#PLTCL-GLOBAL)

Às vezes, é útil ter alguns dados globais que são mantidos entre duas chamadas a uma função ou que são compartilhados entre diferentes funções. Isso é facilmente feito em PL/Tcl, mas há algumas restrições que devem ser compreendidas.

Por razões de segurança, o PL/Tcl executa funções chamadas por qualquer um dos papéis SQL em um interpretador Tcl separado para esse papel. Isso previne interferência acidental ou maliciosa de um usuário no comportamento das funções PL/Tcl de outro usuário. Cada interpretador terá seus próprios valores para quaisquer variáveis Tcl “globais”. Assim, duas funções PL/Tcl compartilharão as mesmas variáveis globais se e somente se forem executadas pelo mesmo papel SQL. Em uma aplicação em que uma única sessão executa código sob múltiplos papéis SQL (por meio das funções `SECURITY DEFINER`, uso de `SET ROLE`, etc.), você pode precisar tomar medidas explícitas para garantir que as funções PL/Tcl possam compartilhar dados. Para isso, certifique-se de que as funções que devem se comunicar sejam de propriedade do mesmo usuário e marque-as `SECURITY DEFINER`. Você deve, naturalmente, ter cuidado para que tais funções não possam ser usadas para fazer algo não intencional.

Todas as funções PL/TclU usadas em uma sessão são executadas no mesmo interpretador Tcl, que, claro, é distinto dos(s) interpretador(es) usado(s) para as funções PL/Tcl. Portanto, os dados globais são compartilhados automaticamente entre as funções PL/TclU. Isso não é considerado um risco de segurança porque todas as funções PL/TclU são executadas no mesmo nível de confiança, ou seja, o nível de um superusuário de banco de dados.

Para ajudar a proteger as funções PL/Tcl de interferirem inadvertidamente umas nas outras, uma matriz global é disponibilizada a cada função através do comando `upvar`. O nome global dessa variável é o nome interno da função, e o nome local é `GD`. Recomenda-se que `GD` seja usado para dados privados persistentes de uma função. Use apenas variáveis globais regulares de Tcl para valores que você especificamente pretende compartilhar entre várias funções. (Observe que os arrays `GD` são apenas globais dentro de um interpretador particular, portanto, eles não contornam as restrições de segurança mencionadas acima.)
