## 33.1. Introdução [#](#LO-INTRO)

Todos os objetos grandes são armazenados em uma única tabela do sistema denominada `pg_largeobject`(catalog-pg-largeobject.md "52.30. pg_largeobject"). Cada objeto grande também tem uma entrada na tabela do sistema `pg_largeobject_metadata`(catalog-pg-largeobject-metadata.md "52.31. pg_largeobject_metadata"). Objetos grandes podem ser criados, modificados e excluídos usando uma API de leitura/escrita que é semelhante às operações padrão em arquivos.

O PostgreSQL também suporta um sistema de armazenamento chamado [“TOAST”](storage-toast.md), que armazena automaticamente valores maiores do que uma única página de banco de dados em uma área de armazenamento secundária por tabela. Isso torna a facilidade de objeto grande parcialmente obsoleta. Uma vantagem restante da facilidade de objeto grande é que ela permite valores de até 4 TB de tamanho, enquanto os campos TOAST podem ter no máximo 1 GB. Além disso, é possível ler e atualizar partes de um objeto grande de forma eficiente, enquanto a maioria das operações em um campo TOAST lerá ou escreverá o valor inteiro como uma unidade.
