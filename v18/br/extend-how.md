## 36.1. Como funciona a extensibilidade [#](#EXTEND-HOW)

O PostgreSQL é extensivo porque sua operação é orientada por catálogo. Se você está familiarizado com sistemas padrão de bancos de dados relacionais, sabe que eles armazenam informações sobre bancos de dados, tabelas, colunas, etc., nos que são comumente conhecidos como catálogos do sistema. (Alguns sistemas chamam isso de dicionário de dados.) Os catálogos aparecem ao usuário como tabelas como qualquer outra, mas o DBMS armazena sua contabilidade interna neles. Uma diferença chave entre o PostgreSQL e os sistemas de banco de dados relacionais padrão é que o PostgreSQL armazena muito mais informações em seus catálogos: não apenas informações sobre tabelas e colunas, mas também informações sobre tipos de dados, funções, métodos de acesso, e assim por diante. Essas tabelas podem ser modificadas pelo usuário, e como o PostgreSQL baseia sua operação nessas tabelas, isso significa que o PostgreSQL pode ser estendido por usuários. Em comparação, os sistemas de banco de dados convencionais só podem ser estendidos alterando procedimentos codificados em linguagem de programação no código fonte ou carregando módulos especialmente escritos pelo fornecedor do DBMS.

O servidor PostgreSQL pode, além disso, incorporar código escrito pelo usuário em si, através de carregamento dinâmico. Isso significa que o usuário pode especificar um arquivo de código objeto (por exemplo, uma biblioteca compartilhada) que implemente um novo tipo ou função, e o PostgreSQL o carregará conforme necessário. O código escrito em SQL é ainda mais trivial de adicionar ao servidor. Essa capacidade de modificar sua operação “on the fly” torna o PostgreSQL excepcionalmente adequado para prototipagem rápida de novas aplicações e estruturas de armazenamento.
