## 13.6. Avisos [#](#MVCC-CAVEATS)

Alguns comandos DDL, atualmente apenas `TRUNCATE` e as formas de reescrita de tabela dos `ALTER TABLE` (sql-altertable.md "ALTER TABLE"), não são seguras para MVCC. Isso significa que, após os commits de truncação ou reescrita, a tabela parecerá vazia para transações concorrentes, se elas estiverem usando um snapshot tirado antes do comando DDL ser comprometido. Isso será um problema apenas para uma transação que não tenha acessado a tabela em questão antes do início do comando DDL — qualquer transação que tenha feito isso manteria pelo menos um bloqueio de tabela `ACCESS SHARE`, o que bloquearia o comando DDL até que essa transação seja concluída. Portanto, esses comandos não causarão nenhuma inconsistência aparente nos conteúdos da tabela para consultas sucessivas na tabela alvo, mas poderiam causar inconsistência visível entre os conteúdos da tabela alvo e outras tabelas no banco de dados.

O suporte ao nível de isolamento de transação Serializable ainda não foi adicionado aos alvos de replicação em standby quente (descrito em [Seção 26.4][(hot-standby.md "26.4. Hot Standby")]). O nível de isolamento mais rigoroso atualmente suportado no modo standby quente é o Repeatable Read. Embora realizar todas as escritas permanentes no banco de dados dentro das transações Serializable no primário garanta que todos os standby eventualmente atinjam um estado consistente, uma transação Repeatable Read executada no standby pode, às vezes, ver um estado transitório que é inconsistente com qualquer execução serial das transações no primário.

O acesso interno aos catálogos do sistema não é feito usando o nível de isolamento da transação atual. Isso significa que objetos de banco de dados recém-criados, como tabelas, são visíveis para transações concorrentes de leitura repetitiva e serializável, mesmo que as linhas que contêm não sejam. Em contraste, as consultas que examinam explicitamente os catálogos do sistema não veem linhas que representam objetos de banco de dados criados simultaneamente, nos níveis de isolamento mais altos.
