## 21.6. Segurança da função [#](#PERM-FUNCTIONS)

As funções, gatilhos e políticas de segurança em nível de linha permitem que os usuários inseram código no servidor do backend que outros usuários possam executar sem intenção. Assim, esses mecanismos permitem que os usuários "troquem cavalos de troia" com relativa facilidade. A proteção mais forte é o controle rigoroso sobre quem pode definir objetos. Onde isso é inviável, escreva consultas que se referem apenas a objetos com proprietários de confiança. Remova dos `search_path` quaisquer esquemas que permitam que usuários não confiáveis criem objetos.

As funções são executadas dentro do processo do servidor de backend com as permissões do sistema operacional do daemon do servidor de banco de dados. Se a linguagem de programação usada para a função permitir acessos de memória não verificados, é possível alterar as estruturas de dados internas do servidor. Assim, entre muitas outras coisas, tais funções podem contornar quaisquer controles de acesso do sistema. As linguagens de função que permitem tais acessos são consideradas “não confiáveis”, e o PostgreSQL permite que apenas superusuários criem funções escritas nessas linguagens.
