### 5.13. Dados estrangeiros [#](#DDL-FOREIGN-DATA)

O PostgreSQL implementa partes da especificação SQL/MED, permitindo que você acesse dados que residem fora do PostgreSQL usando consultas SQL regulares. Esses dados são referenciados como *dados externos*. (Observe que esse uso não deve ser confundido com chaves externas, que são um tipo de restrição dentro do banco de dados.)

Os dados externos são acessados com a ajuda de um *wrapper de dados externos*. Um wrapper de dados externos é uma biblioteca que pode se comunicar com uma fonte de dados externa, ocultando os detalhes de conexão com a fonte de dados e a obtenção de dados dela. Existem alguns wrappers de dados externos disponíveis como módulos `contrib`; veja [Apêndice F](contrib.md "Appendix F. Additional Supplied Modules and Extensions"). Outros tipos de wrappers de dados externos podem ser encontrados como produtos de terceiros. Se nenhum dos wrappers de dados externos existentes atender às suas necessidades, você pode criar o seu próprio; veja [Capítulo 58](fdwhandler.md "Chapter 58. Writing a Foreign Data Wrapper").

Para acessar dados externos, você precisa criar um objeto *servidor externo*, que define como se conectar a uma fonte de dados externa específica de acordo com o conjunto de opções usadas pelo seu wrapper de dados externos de suporte. Em seguida, você precisa criar uma ou mais *tabelas externas*, que definem a estrutura dos dados remotos. Uma tabela externa pode ser usada em consultas da mesma forma que uma tabela normal, mas uma tabela externa não tem armazenamento no servidor PostgreSQL. Sempre que é usada, o PostgreSQL pede ao wrapper de dados externos para obter dados da fonte externa ou transmitir dados para a fonte externa no caso de comandos de atualização.

Para acessar dados remotos, pode ser necessário autenticar-se na fonte de dados externa. Essas informações podem ser fornecidas por um *mapeamento de usuário*, que pode fornecer dados adicionais, como nomes de usuário e senhas, com base no papel atual do PostgreSQL.

Para obter informações adicionais, consulte [CREATE FOREIGN DATA WRAPPER](sql-createforeigndatawrapper.md "CREATE FOREIGN DATA WRAPPER"), [CREATE SERVER](sql-createserver.md "CREATE SERVER"), [CREATE USER MAPPING](sql-createusermapping.md "CREATE USER MAPPING"), [CREATE FOREIGN TABLE](sql-createforeigntable.md "CREATE FOREIGN TABLE") e [IMPORT FOREIGN SCHEMA](sql-importforeignschema.md "IMPORT FOREIGN SCHEMA").
