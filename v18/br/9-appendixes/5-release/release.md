As notas de lançamento contêm as mudanças significativas em cada lançamento do PostgreSQL, com as principais funcionalidades e problemas de migração listados no topo. As notas de lançamento não contêm mudanças que afetam apenas alguns usuários ou mudanças internas e, portanto, não visíveis para o usuário. Por exemplo, o otimizador é melhorado em quase todos os lançamentos, mas as melhorias são geralmente observadas pelos usuários como simplesmente consultas mais rápidas.

Uma lista completa das alterações para cada versão pode ser obtida ao visualizar os logs do [Git](git.md "I.1. Getting the Source via Git") para cada versão. A lista de e-mail `pgsql-committers` (https://www.postgresql.org/list/pgsql-committers/) também registra todas as alterações do código-fonte. Há também uma [interface web](https://git.postgresql.org/gitweb/?p=postgresql.git;a=summary) que mostra as alterações em arquivos específicos.

O nome que aparece ao lado de cada item representa o principal desenvolvedor desse item. Claro que todas as mudanças envolvem discussão da comunidade e revisão de patches, então cada item é realmente um esforço da comunidade.

Os marcadores de seção (§) nas notas de lançamento estão vinculados às páginas [gitweb](https://git.postgresql.org/gitweb/?p=postgresql.git) que mostram as mensagens de commit primário do git e as mudanças na árvore de origem responsáveis pelo item da nota de lançamento. Pode haver outros commits do git que não são mostrados.
