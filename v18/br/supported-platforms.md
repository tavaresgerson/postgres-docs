## 17.6. Plataformas suportadas [#](#SUPPORTED-PLATFORMS)

Uma plataforma (ou seja, uma combinação de arquitetura de CPU e sistema operacional) é considerada compatível com a comunidade de desenvolvimento do PostgreSQL se o código contiver disposições para trabalhar nessa plataforma e tiver sido recentemente verificado para ser construído e passar pelos seus testes de regressão nessa plataforma. Atualmente, a maioria dos testes de compatibilidade de plataforma é realizada automaticamente por máquinas de teste na [PostgreSQL Build Farm](https://buildfarm.postgresql.org/). Se você está interessado em usar o PostgreSQL em uma plataforma que não está representada na fazenda de construção, mas na qual o código funciona ou pode ser feito para funcionar, é fortemente recomendado que você configure uma máquina membro da fazenda de construção para que a compatibilidade contínua possa ser assegurada.

De modo geral, pode-se esperar que o PostgreSQL funcione nessas arquiteturas de CPU: x86, PowerPC, S/390, SPARC, ARM, MIPS e RISC-V, incluindo as variantes big-endian, little-endian, de 32 bits e de 64 bits, quando aplicável.

Pode-se esperar que o PostgreSQL funcione nas versões atuais desses sistemas operacionais: Linux, Windows, FreeBSD, OpenBSD, NetBSD, DragonFlyBSD, macOS, Solaris e illumos. Outros sistemas semelhantes ao Unix também podem funcionar, mas atualmente não estão sendo testados. Na maioria dos casos, todas as arquiteturas de CPU suportadas por um sistema operacional específico funcionarão. Consulte [Seção 17.7](installation-platform-notes.md) abaixo para ver se há informações específicas para o seu sistema operacional, especialmente se estiver usando um sistema mais antigo.

Se você tiver problemas de instalação em uma plataforma que é conhecida por ser suportada de acordo com os resultados recentes da farm de compilação, informe isso ao `<pgsql-bugs@lists.postgresql.org>`. Se você está interessado em portar o PostgreSQL para uma nova plataforma, `<pgsql-hackers@lists.postgresql.org>` é o lugar apropriado para discutir isso.

Versões históricas do PostgreSQL ou POSTGRES também foram executadas em arquiteturas de CPU, incluindo Alpha, Itanium, M32R, M68K, M88K, NS32K, PA-RISC, SuperH e VAX, e sistemas operacionais, incluindo 4.3BSD, AIX, BEOS, BSD/OS, DG/UX, Dynix, HP-UX, IRIX, NeXTSTEP, QNX, SCO, SINIX, Sprite, SunOS, Tru64 UNIX e ULTRIX.
